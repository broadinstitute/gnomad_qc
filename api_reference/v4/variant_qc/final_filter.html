gnomad_qc.v4.variant_qc.final_filter

Script to create final filter Table for release.

usage: gnomad_qc.v4.variant_qc.final_filter.py [-h]
                                               [--slack-channel SLACK_CHANNEL]
                                               [--overwrite] [--test]
                                               [--model-id MODEL_ID]
                                               [--inbreeding-coeff-threshold INBREEDING_COEFF_THRESHOLD]
                                               (--snv-bin-cutoff SNV_BIN_CUTOFF | --snv-score-cutoff SNV_SCORE_CUTOFF)
                                               [--snv-use-interval-qc-bin]
                                               [--snv-use-calling-interval-bin]
                                               (--indel-bin-cutoff INDEL_BIN_CUTOFF | --indel-score-cutoff INDEL_SCORE_CUTOFF)
                                               [--indel-use-interval-qc-bin]
                                               [--indel-use-calling-interval-bin]
                                               [--all-variants]
                                               [--only-filters]

Named Arguments

--slack-channel

Slack channel to post results and notifications to.

--overwrite

Overwrite all data from this subset (default: False).

Default: False

--test

Whether to run a test using only the first 2 partitions of the variant QC Table.

Default: False

--model-id

Filtering model ID to use.

--inbreeding-coeff-threshold

InbreedingCoeff hard filter to use for variants.

Default: -0.3

--snv-bin-cutoff

RF or VQSR score bin to use as cutoff for SNVs. Value should be between 1 and 100.

--snv-score-cutoff

RF or VQSR score to use as cutoff for SNVs.

--snv-use-interval-qc-bin

Whether the ‘–snv-bin-cutoff’ should be applied to the interval QC bin instead of overall bin.

Default: False

--snv-use-calling-interval-bin

Whether the ‘–snv-bin-cutoff’ should be applied to the calling interval bin instead of overall bin.

Default: False

--indel-bin-cutoff

RF or VQSR score bin to use as cutoff for indels. Value should be between 1 and 100.

--indel-score-cutoff

RF or VQSR score to use as cutoff for indels.

--indel-use-interval-qc-bin

Whether the ‘–indel-bin-cutoff’ should be applied to the interval QC bin instead of overall bin.

Default: False

--indel-use-calling-interval-bin

Whether the ‘–indel-bin-cutoff’ should be applied to the calling interval bin instead of overall bin.

Default: False

--all-variants

Whether to write out a version of the final filter Table with all variants, including those not in the release.

Default: False

--only-filters

Whether to write out a version of the final filter Table with only the ‘filters’ field.

Default: False

gnomad_qc.v4.variant_qc.final_filter.process_score_cutoffs(ht, snv_bin_cutoff=None, indel_bin_cutoff=None, snv_score_cutoff=None, indel_score_cutoff=None, aggregated_bin_ht=None, snv_bin_id=None, indel_bin_id=None)[source]

Determine SNP and indel score cutoffs if given bin instead of score.

Note

  • snv_bin_cutoff and snv_score_cutoff are mutually exclusive, and one must be supplied.

  • indel_bin_cutoff and indel_score_cutoff are mutually exclusive, and one must be supplied.

  • If a snv_bin_cutoff or indel_bin_cutoff cutoff is supplied then an aggregated_bin_ht and bin_id must also be supplied to determine the SNP and indel scores to use as cutoffs from an aggregated bin Table like one created by compute_grouped_binned_ht in combination with score_bin_agg.

Parameters:

  • ht (Table) – Filtering Table to prepare as the final filter Table.

  • snv_bin_cutoff (int) – Bin cutoff to use for SNP variant QC filter. Can’t be used with snv_score_cutoff.

  • indel_bin_cutoff (int) – Bin cutoff to use for indel variant QC filter. Can’t be used with indel_score_cutoff.

  • snv_score_cutoff (float) – Score cutoff (e.g. RF probability or AS_VQSLOD) to use for SNP variant QC filter. Can’t be used with snv_bin_cutoff.

  • indel_score_cutoff (float) – Score cutoff (e.g. RF probability or AS_VQSLOD) to use for indel variant QC filter. Can’t be used with indel_bin_cutoff.

  • aggregated_bin_ht (Optional[Table]) – Table with aggregate counts of variants based on bins

  • snv_bin_id (Optional[str]) – Name of bin to use in ‘bin_id’ column of aggregated_bin_ht to determine the SNP score cutoff.

  • indel_bin_id (Optional[str]) – Name of bin to use in ‘bin_id’ column of aggregated_bin_ht to determine the indel score cutoff.

Return type:

Dict[str, StructExpression]

Returns:

Finalized random forest Table annotated with variant filters.

gnomad_qc.v4.variant_qc.final_filter.generate_final_filter_ht(ht, filter_name, score_name, freq_expr, score_cutoff_globals, mono_allelic_flag=True, only_het_flag=True, inbreeding_coeff_cutoff=-0.3)[source]

Prepare finalized filtering model given a filtering HT and cutoffs for filtering.

Parameters:

  • ht (Table) – Filtering Table to prepare as the final filter Table.

  • filter_name (str) – Filtering model name to use in the ‘filters’ field (AS_VQSR or RF).

  • score_name (str) – Name to use for the filtering score annotation. This will be used in place of ‘score’ in the release HT info struct and the INFO field of the VCF (e.g. RF or AS_VQSLOD).

  • freq_expr (ArrayExpression) – Expression for frequency information to use for flags and filters.

  • score_cutoff_globals (Dict[str, StructExpression]) – Dictionary of score cutoffs to use for filtering.

  • mono_allelic_flag (bool) – Whether to add a flag indicating that a variant is mono-allelic.

  • only_het_flag (bool) – Whether to add a flag indicating that all carriers of a variant are het.

  • inbreeding_coeff_cutoff (float) – InbreedingCoeff hard filter to use for variants.

Return type:

Table

Returns:

Finalized random forest Table annotated with variant filters.
