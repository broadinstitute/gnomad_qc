gnomad_qc.v4.sample_qc.outlier_filtering.apply_regressed_filtering_method(sample_qc_ht, qc_metrics, pop_scores_expr=None, platform_scores_expr=None, platform_expr=None, regress_pop_n_pcs=30, regress_platform_n_pcs=9, regress_per_platform=False, include_unreleasable_in_regression=False, include_unreleasable_in_cutoffs=False)[source]ÔÉÅ

Compute sample QC metrics residuals after regressing out specified PCs and determine what samples are outliers that should be filtered.

The following are all possible filtering options:

  • Include pop_scores_expr: Regress population PCs only and determine outliers for each metric in qc_metrics.

  • Include platform_scores_expr: Regress platform PCs only and determine outliers for each metric in qc_metrics.

  • Include pop_scores_expr and platform_scores_expr: Regress both population PCs and platform PCs and determine outliers for each metric in qc_metrics.

  • For all of the above filtering options, there is also a regress_per_platform option, which will stratify the dataset by platform before performing the regression and outlier filtering.

After regression, compute_stratified_metrics_filter is used to compute the median, MAD, and upper and lower thresholds for each of the qc_metrics residuals, optionally stratified by assigned platform, and return a qc_metrics_filters annotation indicating if the sample falls a certain number of MAD outside the distribution.

Note

Default left and right MAD for compute_stratified_metrics_filter is 4. We modify this for ‚Äòn_singleton_residual‚Äô to be (math.inf, 8.0) and for ‚Äòr_het_hom_var_residual‚Äô to be (math.inf, 4.0). The math.inf (infinity) is used to prevent a lower cutoff for these metrics.

Parameters:

  • sample_qc_ht (Table) ‚Äì Sample QC HT.

  • qc_metrics (List[str]) ‚Äì Specific metrics to use for outlier detection.

  • pop_scores_expr (Optional[ArrayExpression]) ‚Äì Optional expression with population PCA scores.

  • platform_scores_expr (Optional[ArrayExpression]) ‚Äì Optional expression with platform PCA scores.

  • platform_expr (Optional[StringExpression]) ‚Äì Optional expression with platform assignment.

  • regress_pop_n_pcs (Optional[int]) ‚Äì Number of population PCA scores to use in regression. Default is 30.

  • regress_platform_n_pcs (Optional[int]) ‚Äì Number of platform PCA scores to use in regression. Default is 9.

  • regress_per_platform (bool) ‚Äì Whether to perform the regression per platform. Default is False.

  • include_unreleasable_in_regression (bool) ‚Äì Whether to include unreleasable samples in the regressions.

  • include_unreleasable_in_cutoffs (bool) ‚Äì Whether to include unreleasable samples in the determination of filtering cutoffs.

Return type:

gnomad_qc.v4.sample_qc.outlier_filtering.apply_n_singleton_filter_to_r_ti_tv_singleton(ht, sample_qc_ht, filtering_method, ann_exprs, **kwargs)[source]ÔÉÅ

Apply n_singleton median sample filter and update the r_ti_tv_singleton outlier filtering.

This function takes a Table (ht) returned from one of the filtering functions:

  • apply_stratified_filtering_method

  • apply_regressed_filtering_method

  • apply_nearest_neighbor_filtering_method

which must include filtering information for ‚Äòn_singleton‚Äô or ‚Äòn_singleton_residuals‚Äô.

Median values for ‚Äòn_singleton‚Äô or ‚Äòn_singleton_residuals‚Äô are extracted from ‚Äòqc_metrics_stats‚Äô in ht. ‚Äòqc_metrics_stats‚Äô takes different forms depending on filtering_method and the use of ‚Äòstrata‚Äô (defined as a global on ht if used) in filtering (described below).

Then the sample_qc_ht is filtered to only samples with ‚Äòn_singleton‚Äô (or ‚Äòn_singleton_residuals‚Äô) > the median in ‚Äòqc_metrics_stats‚Äô that is relevant to the sample. This filtered sample_qc_ht is rerun through the filtering function for filtering_method using only ‚Äòr_ti_tv_singleton‚Äô, and ht is updated with new global and row values relevant to ‚Äòr_ti_tv_singleton‚Äô filtering.

If filtering_method is ‚Äòstratified‚Äô:

  • ‚Äòstrata‚Äô is defined as a global annotation on ht in the form: tuple<str, ...> where the number of string elements in the tuple depends on the number of strata.

  • ‚Äòqc_metrics_stats‚Äô is defined as a global annotation on ht in the form:

dict<strata, struct {
    metric: struct {
        'median': float64, 'mad': float64, 'lower': float64, 'upper': float64
    }
}>

If filtering_method is ‚Äòregressed‚Äô:

  • ‚Äòqc_metrics_stats‚Äô is defined as a global annotation on ht.

  • If platform stratification was performed, the ‚Äòstrata‚Äô global annotation is tuple<'platform'>, and ‚Äòqc_metrics_stats‚Äô is in the same form as described for ‚Äòstratified‚Äô, except metric -> metric_residual.

  • If there was no platform stratification, ‚Äòqc_metrics_stats‚Äô is in the form:

struct {
    metric_residual: struct {
        'median': float64, 'mad': float64, 'lower': float64, 'upper': float64
    }
}

If filtering_method is ‚Äònearest_neighbor‚Äô:

  • ‚Äòqc_metrics_stats‚Äô is defined as a row annotation on ht in the form:

struct {
    metric: struct {
    'median': float64, 'mad': float64, 'lower': float64, 'upper': float64
    }
}
Parameters:

  • ht (Table) ‚Äì Input filtering Table with n_singleton or n_singleton_residuals medians computed.

  • sample_qc_ht (Table) ‚Äì Sample QC HT.

  • filtering_method (str) ‚Äì The filtering method to apply to sample_qc_ht. One of ‚Äòstratified‚Äô, ‚Äòregressed‚Äô, or ‚Äònearest_neighbors‚Äô.

  • ann_exprs (Dict[str, Expression]) ‚Äì Dictionary of the expressions for population assignment and/or platform assignment on sample_qc_ht to be used if filtering is stratified.

  • kwargs (Any) ‚Äì

Return type:

Table

Table with outlier filter annotations updated for r_ti_tv_singleton or r_ti_tv_singleton_residuals.

gnomad_qc.v4.sample_qc.outlier_filtering.create_finalized_outlier_filter_ht(finalized_outlier_hts, qc_metrics, ensemble_operator='and')[source]ÔÉÅ

Create the finalized outlier filtering Table.

Note

qc_metrics must be the same as, or a subset of the original QC metrics used to create each Table in finalized_outlier_hts.

Tables included in finalized_outlier_hts are reformatted to include only information from metrics in`qc_metrics`, and the ‚Äòqc_metrics_filters‚Äô annotation is modified to include only metrics in qc_metrics.

If finalized_outlier_hts includes multiple Tables, the reformatted Table annotations are grouped under top level annotations specified by the keys of finalized_outlier_hts. The following annotations are also added:

  • qc_metrics_fail: includes the combined (using ensemble_operator) outlier fail boolean for each metric.

  • qc_metrics_filters: The combined set of QC metrics that a sample is an outlier for using ensemble_operator for the combination (‚Äòand‚Äô uses set intersection, ‚Äòor‚Äô uses set union).

Finally, an outlier_filtered annotation is added indicating if a sample has any metrics in qc_metrics_filters.

Parameters:

  • finalized_outlier_hts (Dict[str, Table]) ‚Äì Dictionary containing Tables to use for the finalized outlier filtering, keyed by a string to use as a top level annotation to group the annotations and global annotations under.

  • qc_metrics (List[str]) ‚Äì List of sample QC metrics to use for the finalized outlier filtering.

  • ensemble_operator (str) ‚Äì Logical operator to use for combining filtering methods when multiple Tables are in finalized_outlier_hts. Options are [‚Äòor‚Äô, ‚Äòand‚Äô]. Default is ‚Äòand‚Äô.

Return type:

Table

Returns:

Finalized outlier filtering Table.
